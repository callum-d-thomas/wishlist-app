# Project Brief

## Project Name
Wishlist App

## Vision
A web application for managing shared wishlists for private groups, families, and friends, helping people coordinate gift-giving for special occasions while avoiding duplicate purchases.

## Core Problem
When planning gifts for special occasions (birthdays, holidays, weddings), people often:
- Don't know what others actually want
- Risk buying duplicate gifts
- Lack a private way to coordinate with family/friends
- Find existing solutions too complex or expensive

## Solution
A privacy-first wishlist app where:
- Users create wishlists for occasions
- Invite specific people to private groups
- Members can "claim" items (hidden from others) to prevent duplicates
- Self-hostable for cost efficiency

## Target Users
- Families coordinating holiday/birthday gifts
- Friend groups planning for weddings, baby showers, etc.
- Small private groups who value privacy and control

## Key Differentiators
1. **Privacy-First**: Private group wishlists, not social media
2. **Claiming System**: Prevent duplicate gifts effectively
3. **Self-Hostable**: Run on personal infrastructure, low cost
4. **Learning Project**: Opportunity to master modern tech stack

## Success Criteria
- Users can create and share wishlists within 5 minutes
- Claiming system prevents duplicate gifts 100% of the time
- Self-hostable on personal infrastructure with minimal setup
- Code quality demonstrates mastery of modern patterns (DDD, SOLID)
- 85%+ test coverage for critical features

## Out of Scope (Initially)
- Public/collaborative wishlists
- Price tracking or web scraping
- Mobile native apps
- Social media integration
- Advanced analytics

## Technology Choices
- **Frontend**: Angular 21+ (learning focus), Bootstrap 5.3.8+
- **Backend**: ASP.NET Core 10 (learning focus)
- **Database**: SQL Server Express (self-hostable)
- **Hosting**: Personal infrastructure (cost efficiency)

## Timeline
This is a learning project without hard deadlines. Focus on quality over speed.
